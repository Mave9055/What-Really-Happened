# WRH Documentation Architecture Standard

## Purpose

This document defines the required structure for all WRH repositories to ensure clarity, auditability, and safe institutional deployment.

WRH is a facilitator-led trauma education system designed for structured group delivery. All documentation must support safe implementation, clear boundaries, and measurable outcomes.

---

## Core Documentation Layers

All WRH repositories should include the following layers.

### 1. Program Identity Layer
- README.md (system overview and purpose)
- LICENSE (usage and distribution terms)

### 2. Clinical & Theoretical Layer
- EVIDENCE_BASE.md
- SAFETY_AND_ESCALATION.md
- ACCESSIBILITY.md

### 3. Implementation Layer
- IMPLEMENTATION_GUIDE.md
- DEPLOYMENT_CHECKLIST.md
- SESSION_PACKAGES/

### 4. Outcomes Layer
- OUTCOMES_AND_MEASURES.md

### 5. Versioning Layer
- Git tags (`v1.0.0` baseline required for pilot readiness)
- GitHub Releases with human-readable change summaries

---

## Documentation Principles

All WRH documentation must adhere to:

- Plain language over academic abstraction
- Clear boundaries between education and clinical treatment
- Explicit inclusion/exclusion criteria
- Facilitator-first usability
- Audit-ready structure (traceable, versioned, and reviewable)

---

## Safety Principle

WRH content is psychoeducational and facilitator-led. It does not replace clinical care.

All materials must clearly state:

> WRH supports clinical environments but does not function as a standalone clinical intervention.
